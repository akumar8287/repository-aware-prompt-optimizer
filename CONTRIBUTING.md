# Contributing

## How to contribute

1. Fork the repository.
2. Create a branch from `main` in your fork.
3. Make your changes.
4. Open a pull request against `main` in the original repository.
5. All PRs require review and approval from the maintainer before merging.

## What will be rejected

- Changes that remove or modify copyright notices, LICENSE, or NOTICE files.
- Changes that remove attribution from SKILL.md, README.md, or docs.
- Changes that alter plugin.json author/repository fields.
- PRs that attempt to rebrand or remove original authorship.

## What is welcome

- Bug fixes in examples or documentation.
- New example files under `skills/repository-aware-prompt-optimizer/examples/`.
- New task type entries in `references/task-classification.md`.
- Improvements to `tests/evaluation-checklist.md`.
- Translations or additional Hinglish/regional language examples.
